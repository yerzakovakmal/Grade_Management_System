## 1. Модификаторы доступа

Они определяют, **кто может видеть и изменять данные**.

Аналогия: дом

* public — гостиная (доступна всем)
* private — спальня (только тебе)
* protected — семейная комната (только “своим”)

---

### `private`

Доступ только внутри класса.

```cpp
class User {
private:
    static int totalUsers;
};
```

**Почему используем:**
Счётчики нельзя менять извне — иначе данные сломаются.

---

### `protected`

Доступ внутри класса и его наследников.

```cpp
class User {
protected:
    string name;
};
```

**Почему используем:**
Наследники (Student, Professor) могут напрямую использовать `name`.

---

### `public`

Доступен всем.

```cpp
class User {
public:
    string getName() const;
};
```

**Почему:**
Это “интерфейс” класса — то, что используется снаружи.

---

### Правило

* Данные → `private` или `protected`
* Методы для внешнего использования → `public`

## 2. Virtual

---

### Виртуальные функции

Позволяют вызывать **правильную версию функции во время выполнения**.

```cpp
User* u = new Student(...);
u->displayInfo();  // вызовет Student::displayInfo()
```

Это называется **полиморфизм**.

---

### Виртуальный деструктор

```cpp
virtual ~User();
```

**Почему важно:**

```cpp
User* u = new Student();
delete u;
```

Без `virtual` → вызывается только `User`
С `virtual` → сначала `Student`, потом `User`

---

### Виртуальное наследование

Решает **ромбовидную проблему (diamond problem)**.

```cpp
class Student : virtual public User {};
class Professor : virtual public User {};
```

Теперь у `TeachingAssistant` только **один User**, а не два.

## 3. Static

Статические члены принадлежат **классу, а не объекту**.

---

### Статические поля

```cpp
static int nextOrderNum;
```

Общее значение для всех объектов.

---

### Статические методы

```cpp
static int getTotalUsers();
```

Вызываются так:

```cpp
User::getTotalUsers();
```

---

### Важно

Нужно объявить в `.cpp`:

```cpp
int Student::nextOrderNum = 1;
```

---

## 4. Чисто виртуальные функции

```cpp
virtual string getRole() const = 0;
```

Означает:
“Я объявляю функцию, но не реализую её.”

---

### Абстрактный класс

```cpp
User u; // ОШИБКА
```

Нельзя создавать объект.

---

**Почему:**
“Просто пользователь” не существует — только Student, Professor и т.д.

---

## 5. Override

```cpp
string getRole() const override;
```

Проверяет, что функция действительно переопределяет базовую.

---

**Без override:**

Ошибка может остаться незамеченной.

**С override:**

Компилятор сразу покажет проблему.

## 6. Конструкторы и деструкторы

---

### Конструктор

```cpp
User::User(...) : name(name) {}
```

Используется **initializer list** (быстрее).

---

### Деструктор

```cpp
~User() {
    --totalUsers;
}
```

Освобождает ресурсы.

---

### TeachingAssistant

Самый сложный случай:

```cpp
: User(...),
  Student(...),
  Professor(...)
```

Только `TeachingAssistant` вызывает `User`.

## 7. Перегрузка операторов

---

### `==`

```cpp
return userId == other.userId;
```

Сравнение по уникальному ID.

---

### `!=`

```cpp
return !(*this == other);
```

---

### `<<`

```cpp
cout << user;
```

Позволяет красиво выводить объект.

---

### `<`

Используется для сортировки.

---

### `>`

Сравнение оценок.
